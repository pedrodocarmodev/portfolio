Employee API

API responsável por armazenar informações de empregados de uma empresa fictícia. Possui funcionalidades como contratação e demissão de funcionários.

Ver no GitHub

A stack utilizada foi: Go, Gin e PostgreSQL. O projeto consolidou conceitos como Arquitetura em Camadas e integração com banco de dados.

Ordem de dependências: HTTP → Handler → Service → Repository → PostgreSQL

Endpoints

  • GET /employees
  • GET /employees/:id
  • POST /employees
  • PATCH /employees/:id/fire
  • PATCH /employees/:id/employ
