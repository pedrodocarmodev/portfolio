Post Manager

API desenvolvida com o objetivo de gerenciar posts em um blog e comentários relacionados.

Ver no GitHub

A stack utilizada foi: FastAPI, Sqlachemy, Sqlite. O projeto consolidou conceitos de Foreign Key e integração com banco de dados. Entendi melhor como criar e organizar Endpoints.

Arquitetura

routers/ → HTTP layer
services/ → Business logic
repositories/ → Data access layer
schemas/ → Pydantic models
core/ → Database configuration
models/ → SQLAlchemy models

Endpoints

  • POST /posts – Create post
  • GET /posts/{post_id} – Get post by ID
  • PUT /posts/{post_id} – Update post
  • DELETE /posts/{post_id} – Delete post
  • POST /posts/{post_id}/comments – Create comment for a post
  • GET /posts/{post_id}/comments – List comments of a post

Comments

  • GET /comments/{comment_id} – Get comment by ID
  • PUT /comments/{comment_id} – Update comment
  • DELETE /comments/{comment_id} – Delete comment
